# 領収明細票 自動発行・監視システム (Pharmacy Barcode Print System)

調剤薬局向けの領収明細票（薬局控)自動発行およびバックグラウンド監視システムです。
レセコンから出力されるNSIPS（CSV）データをリアルタイムで監視し、POSシステムとの連携に必要なバーコード・QRコードを自動生成してプリンタへ即時出力します。

## 概要 (Overview)
本システムは、薬局の窓口業務を止めない「完全防弾・ノンストップ運用」を前提に設計されています。
Windowsのタスクトレイに常駐し、ネットワーク切断からの自動復旧、エラー時の安全なフォールバック、ファイルロックの回避など、現場特有のトラブルを吸収する堅牢なアーキテクチャを備えています。QRコードのデータ構造はCR+LF改行に最適化されており、POS側の正確な読み取りを担保します。
レジストリを汚さない、ポータブル環境での運用も可能です。

## 主な機能 (Features)
* **NSIPSデータの完全自動監視・解析**: 指定フォルダにA/Uファイルが出現した瞬間に自動パージ・解析を実行。
* **柔軟なバーコード・QR印字**: 1D（EAN13）と2D（QRコード）の並列・単独出力に対応。A6用紙および80mmロール紙（Epson TM-m30II-H等のレシートプリンタ）のレイアウトを動的に切り替え可能。
* **タスクトレイ常駐型ステータスモニター**: 稼働状態（オンライン/オフライン）をアイコンの色で視覚的に通知し、通信切断時はWindowsトースト通知で現場へ即時アラート。
* **安全なGUI設定・再発行ツール**: 現場スタッフが直感的に操作できるTkinterベースの設定エディタと、過去の処方データを瞬時に検索・再発行できる専用ツールを同梱。
* **マルチスレッド＆フェイルセーフ**: ファイル処理、GUI描画、監視ループを完全に分離し、どれか一つがクラッシュしても全体が落ちないデッドロック防止設計。

## 構成モジュール (Architecture)
* `barcode_monitor.py`: システムの中核。常駐してフォルダを監視し、印刷をキックするメインサービス。
* `config_editor_v2.py`: 座標や印字項目、システム挙動を管理するGUI設定ツール。
* `reissue_tool_pro.py`: 履歴データ（JSON）から過去の明細を瞬時に再発行するGUIツール。
* `db_bulk_register.py`: 過去のNSIPSデータを一括でデータベース（JSON）へ取り込む保守用スクリプト。
* `lib_*.py`: 共通処理、バーコード生成描画、NSIPS構文解析などを担うコアライブラリ群。

## 動作環境 (Requirements)
* OS: Windows 10 / 11
* Python: 3.11+
* 主要ライブラリ:
  * `Pillow` (画像処理・バーコード描画)
  * `pystray` (タスクトレイ常駐制御)
  * `python-barcode` / `qrcode`
  * `pywin32` (win32print, win32ui / Windowsネイティブ印刷制御)

## インストールと起動 (Getting Started)
1. リポジトリをクローンします。
2. 依存ライブラリをインストールします。
   ```bash
   pip install -r requirements.txt
3. 初回起動時は config_editor_v2.py を実行し、監視フォルダ（WatchDir）やプリンタ名を設定して barcode_setting.ini を生成します。

4.barcode_monitor.py を実行すると、タスクトレイに緑色のアイコンが出現し、監視がスタートします。

##セキュリティ・運用上の注意 (Security & Operations)
本システムは個人情報（氏名、処方内容等）を取り扱うため、リポジトリの .gitignore にて history/*.json および log/*.txt が除外設定されています。
本番環境で運用する際は、各端末の足元に生成される一時ファイルディレクトリ（work_dir / pdf_output）の取り扱いや、アクセス権限の管理に十分注意してください。
